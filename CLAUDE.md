# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

Threshold is a local-first, trauma-informed AI assistant for people navigating re-entry after incarceration. Built with deepagents, LangGraph, and Claude. Python 3.12+, managed with `uv`.

## Commands

```bash
uv sync                                   # Install dependencies
uv run python main.py seed                # Create test profile (generates encryption key + demo user "Tyler")
uv run python main.py chat                # Start interactive chat
uv run python main.py profile             # View current profile
uv run python -m threshold.voice.bot      # Start voice interview bot (serves client at localhost:7860/client)
langgraph dev                             # Run with LangGraph Studio visual debugger
```

## Architecture

**Orchestrator pattern via deepagents:** The main agent (`threshold/agents/orchestrator.py`) uses `create_deep_agent()` from `deepagents`. It routes to subagents via the built-in `task()` tool â€” no manual routing logic needed. The orchestrator decides based on subagent descriptions.

**Subagents** are plain Python dicts in `threshold/agents/subagents/` with keys: `name`, `description`, `system_prompt`, `tools`, `model`. Currently: `benefits` (Anthropic Haiku), `employment` (Anthropic Sonnet), `housing` (xAI Grok via ChatOpenAI), `legal` (Anthropic Haiku), `form-filler` (CompiledSubAgent using Anthropic Sonnet + computer use). The orchestrator only keeps crisis + memory tools directly â€” all domain tools are delegated to subagents.

**Form filler** (`threshold/agents/subagents/form_filler.py`) is a `CompiledSubAgent` that uses Anthropic's computer use tool (`computer_20251124`) with a remote Browserbase browser to fill online government forms. The agentic loop is in `threshold/tools/form_filler/loop.py`. It never clicks submit â€” the user reviews via a live view URL and submits manually. Only `.gov` URLs are allowed (see `safety.py`). Sensitive profile fields (offense, supervision) are redacted before sending to forms.

**Tools** are `@tool`-decorated functions (from `langchain_core.tools`) that always return `str`. To add a tool: define it, export from `threshold/tools/__init__.py`, add to the tool list in the orchestrator or relevant subagent, and mention it in the system prompt.

**Memory system:** User profiles are Pydantic models (`UserProfile` in `threshold/memory/profile.py`) encrypted with Fernet (AES-256) and stored at `data/profile/structured_profile.json.enc`. The encryption key is in `THRESHOLD_ENCRYPTION_KEY` env var. `AGENTS.md` serves as the agent's long-term memory file, auto-updated at runtime.

**Workflows** in `workflows/` are markdown step-by-step instructions that agents read at runtime via `read_file()`. Referenced in system prompts. To add a workflow: create `.md` in `workflows/`, reference in the orchestrator system prompt (~line 66 of orchestrator.py).

**CLI** uses Typer + Rich (`main.py`). The chat loop invokes the orchestrator graph with LangGraph's message-based interface.

**Voice interview agent** (`threshold/voice/`) is a Pipecat-powered voice intake pipeline grounded in motivational interviewing (MI). It uses Pipecat Flows for phase management with dynamic NodeConfig generation, an EngagementTracker processor for real-time emotional adaptation, and function tools that save fields to the DB + observations to the memory stream during conversation. The pipeline: SmallWebRTCTransport (peer-to-peer, no API key) â†’ Deepgram STT â†’ EngagementTracker â†’ Anthropic Claude Sonnet (LLM) â†’ ElevenLabs TTS â†’ WebRTC output. Run with `python -m threshold.voice.bot` (serves client at `http://localhost:7860/client`). Post-interview synthesis generates a person-centered summary, highlight reel, and care plan seed. Frontend at `/interview` shows live schema visualization via WebSocket events. API endpoints under `/api/voice/*`.

## Key Constraints

- `search_jobs()` and `search_housing()` are **stubs** returning mock data. They need real API integrations (Adzuna, 211.org, etc.) but function signatures and `@tool` decorator + `str` return type must be preserved.
- The crisis protocol is non-negotiable: if a user expresses crisis, `crisis_response()` must be called immediately with no delegation.
- Tool return format is always `str` with markdown formatting.

## Environment

Requires `ANTHROPIC_API_KEY`, `GOOGLE_API_KEY` (orchestrator), `BROWSER_BASE_API` + `BROWSER_BASE_PROJECT_ID` (form filler), `XAI_API_KEY` (housing subagent), `DEEPGRAM_API_KEY` (voice STT), and `ELEVENLABS_API_KEY` (voice TTS). Voice transport uses SmallWebRTC (no API key needed). `THRESHOLD_ENCRYPTION_KEY` is auto-generated by `seed`. See README.md for full env var table.

**Python version:** Must use Python 3.13 (pinned in `.python-version`). Python 3.14 alpha causes segfaults in native extensions (pydantic-core, orjson) used by langchain.

**Multi-model setup:** The project uses multiple LLM providers:
- Orchestrator + reflection: Google Gemini (`gemini-2.5-flash`) via `ChatGoogleGenerativeAI`
- Benefits, legal subagents: Anthropic (`claude-haiku-4-5-20251001`)
- Employment subagent, form filler loop: Anthropic (`claude-sonnet-4-6`)
- Housing subagent: xAI Grok (`grok-4-1-fast`) via `ChatOpenAI` with custom `base_url`
- Voice interview LLM: Google Gemini (`gemini-2.5-flash`) via Pipecat `GoogleLLMService`
- Voice interview TTS: ElevenLabs (`eleven_turbo_v2_5`) via Pipecat `ElevenLabsTTSService`
- Voice interview STT: Deepgram via Pipecat `DeepgramSTTService`
- Voice interview transport: SmallWebRTCTransport (peer-to-peer, no API key)

## Tests

```bash
uv run python -m unittest discover -s threshold/tests -p 'test_form_filler*.py'
```

Unit tests for the form filler are in `threshold/tests/`. They use mocks/stubs (no live API calls). Cover key normalization, URL allowlisting, profile redaction, the agentic loop, and subagent fill_node behavior.
